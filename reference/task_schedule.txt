

               ##系统任务调度机制##

	任务管理分为：任务触发器/任务分配器/任务调度器/任务管理器。其中任务管理器自带
任务回收，任务状态修改查询，任务优先级插入等功能。

任务调度机制：
-------------------------------------------------------------------------------------------
|   				|-1.管道。
|	    			|    使用有名管道，主要功能是任务触发器通知任务分配器有任务产生。任务通知
|		    		|    标志用一个字符表示（不同字符代表不同动作）。
|	任务触发器		|-2.接受任务。
|			    	|	 系统提供多种连接方式(串口/网络等)，任务由多个渠道产生,将接受的任务挂
|	    			|	 在待处理(pending)队列。
|	    			|	 (任务产生后会通知任务分配器进行处理。)
-------------------------------------------------------------------------------------------

-------------------------------------------------------------------------------------------
|		    		|-1.获取任务。
|		    		|-2.分析任务。
|			    	|    从待处理队列取出任务，分析任务是否为可执行任务，如不是则回收该任务，
|	任务分配器		|   否则继续执行。
|		    		|-3.分配资源。
|		    		|-4.开始任务。
|		    		|	 如果任务可执行，则为其分配必要资源，然后将其状态设置为start交给调度器
-------------------------------------------------------------------------------------------
	   
-------------------------------------------------------------------------------------------
|			    	|-1.调度任务。
|	任务调度器		|   从开始任务队列取出待执行任务，从线程池获取空闲线程开始执行任务。
|		    		|-2.设置任务状态为starting
-------------------------------------------------------------------------------------------

-------------------------------------------------------------------------------------------
|			    	|-1.任务回收。
|	任务管理器		|   当任务已经处理完或者状态为unexcute，则释放任务所占系统空间。
|			    	|-2.任务状态查询。
|			    	|-3.任务插入。
-------------------------------------------------------------------------------------------
